2021 (in progress)

Foundation Directory is a comprehensive database consisting of information of over 239,000 U.S. Foundations. It has a powerful search tool which equips their users with all the tools and strategies for a fastest way to win more funding. Foundation Directory has a collection of paid and free subscription plans. I was responsible for enhancing the user onboarding process for its free plan, Quick Start, so that we can leverage the signed up users from the free plan to paid plans.

My role

  • Rebrand Quick Start, a minified free version of Foundation Directory Professional plan
  • Research and ideate ways to improve the user experience
  • Create prototypes based on user research
  • Style the site in React

Stakeholder interview

The outcome of the interviews informed our understanding of the stakeholders' strategies and the current performance of the site.

Goal briefs:

  • Improve user onboarding process of Quick Start
  • Demonstrate the value of a paid subscription from Quick Start
  • Increase signup ratio in Quick Start for the purpose of user conversion from free to paid plans​​
  • Create a more visually appealing representation of the information​

Challenge

The rebranded site needed to show a sneak peek of what the users could be getting from a paid plan but we shouldn't show any / bare minimum content from the paid plans.

Restrictions:

  1. Instructions were given to not make designs that might need significant dev work on the backend. The amount of dev work needed should be really low.
  2. Do not add any design elements that is visually different from the rest of the content of the page.

I had to find a solution that was easy to implement on the backend + satisfy strategies of demonstrating the value of paid subscription + increase the conversion ratio.

THE PROCESS

Competitive analysis

I studied how other sites were implementing their user onboarding experiences. There was a common pattern of introducing a signup form after interacting with an element in the page.

UI Development

React + Material UI + Storybook

I was responsible for styling the pages in react as well as reviewing styling code from the developers after it was added storybook.

I also worte css code following the CUBE CSS Design System principle, targeting styling code from higher level elements with the purpose reusing them multiple times thus reducing the lines of codes needed.

Key takeaways

  • Communicate with the stakeholders frequently to get their feedback on the design.
  • Get the developers involved in the design meetings so that they are on the loop about any layout changes being made.
  • Create tickets to keep track of process, files and decisions.
  • Iterate considering the point of views of stakeholders and users.
  • Developers need design support so be on the lookout for anything they might need.
  • Have daily standups with the team to know about any problems and progress made
